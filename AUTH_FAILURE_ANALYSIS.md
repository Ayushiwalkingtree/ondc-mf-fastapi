## Why Gateway Returns Authentication Failed

The gateway has moved past Authorization header parsing. It now tries to authenticate the sender.

For ONDC authentication, the gateway must resolve:

- `subscriber_id`: `ondcapi.walkingtree.tech`
- `unique_key_id`: `key-1`
- signing algorithm: `ed25519`
- registered signing public key

Since staging registry lookup currently returns `Subscriber not found`, the gateway cannot resolve or trust the public key for this subscriber/key id. That produces:

```text
401 Unauthorized
Throwable: Authentication failed
```
