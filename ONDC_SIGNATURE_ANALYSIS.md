## Root Cause

The gateway rejects the request with `Header parsing failed` because the `headers` metadata parameter in the Authorization header is malformed.

Current code:

```python
# app/utils/ondc_auth.py:48
'headers="(created)(expires)digest",'
```

Expected ONDC SDK-compatible value:

```python
'headers="(created) (expires) digest",'
```

This is a parser-level failure. The gateway likely fails before checking:

- registry lookup,
- public key,
- digest match,
- Ed25519 signature validity,
- payload schema.

## Notes

- The current digest calculation is compatible with ONDC: `hashlib.blake2b(..., digest_size=64)` and base64 encoding.
- The current signature method is compatible with ONDC: detached Ed25519 signature, base64 encoded.
- `app/utils/json.py` sorts keys before sending. This does not cause the current parser error because `app/services/ondc_client.py` signs the same `raw_body` that it sends.
- After fixing the `headers` metadata value, the gateway may proceed to later checks. Any later NACK will be a separate issue, such as subscriber registry status, key registration, payload schema, BAP URI, or domain readiness.
